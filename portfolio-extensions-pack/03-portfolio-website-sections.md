# Portfolio Website Sections (Recommended)

Use this structure on your portfolio page.

## A) Section: Chrome Extensions (Main Block)

- Section title: Chrome Extensions I Built
- Subtext: Browser tools focused on productivity, visual precision, and capture workflows.
- Cards: 2 cards (one per extension)

### Card Template
- Project name
- One-line value proposition
- 3 key features
- Tech stack chips
- Two CTAs:
  - Live on Chrome Web Store
  - View Case Study / Details

## B) Section: Featured Project Deep Dive

Create one deeper case study block for Advanced Smart Capture.

Include:
- Problem statement
- Why existing tools were fragmented
- Your approach
- Feature architecture summary
- Product and monetization decisions (free + premium)
- Outcome and launch links

## C) Section: UI Precision Tooling

Highlight Pixel Ruler + Alignment Checker as a focused productivity utility.

Include:
- Typical users (front-end devs, designers, QA)
- Workflow example (open page -> measure -> align -> verify)
- Real use cases (spacing audits, responsive checks)

## D) Section: Proof of Work

Add:
- Screenshots gallery
- Chrome Web Store links
- GitHub repo links
- Product Hunt link (for Advanced Smart Capture)

## E) Section: Technical Highlights

Suggested bullet points:
- Chrome Extension Manifest V3 architecture
- Popup + background + content script separation
- Browser-side UX design for high-frequency actions
- Production publishing to Chrome Web Store

## F) Section: CTA

- Primary CTA: Install from Chrome Web Store
- Secondary CTA: Contact me for custom browser tooling
